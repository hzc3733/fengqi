## 1. web（网页搜索与内容提取）

**工具**：`web_search`（网页搜索）、`web_extract`（网页内容提取）

### web_search

**简介**：在互联网上搜索信息，支持 Google、Bing、DuckDuckGo 等搜索引擎。

**核心概念**：
- **搜索引擎（Search Engine）**：提供关键词检索服务的网站，如 Google、Bing、DuckDuckGo。不同引擎的搜索结果排名算法不同。
- **Favicon（Favorite Icon）**：网站左上角的小图标，用于识别网站品牌。
- **Session cookie**：服务器在用户浏览器中存储的小文件，用于保持登录状态和跟踪用户行为。

**常见用法**：
```python
web_search(query="Hermes Agent documentation", source="google", limit=5)
# source 可选: google, bing, ddg, brave, serif, news
```

**返回值**：搜索结果列表，包含标题、URL、摘要。

**注意事项**：
- 不同搜索引擎需要不同的 API Key（如 Google 需要 Google API Key，Bing 需要 Bing API Key）
- DuckDuckGo 通常开箱即用，无需 API Key
- 搜索结果数量建议不超过 10 条，避免 token 浪费

---

### web_extract

**简介**：从指定 URL 提取网页内容，支持正文提取、关键信息定位、增量更新。

**核心概念**：
- **正文提取（Main content extraction）**：从网页 HTML 中识别并提取主体内容，过滤掉导航栏、广告、页脚等噪音内容。
- **CSS 选择器（CSS Selector）**：用于定位 HTML 文档中特定元素的语法，如 `#content`、`div.article > p`。
- **增量更新（Delta/diff extraction）**：只提取网页中与上次相比发生变化的部分，用于监控更新。

**常见用法**：
```python
# 基础提取
web_extract(url="https://example.com/article", query="提取文章标题和主要内容")

# 指定选择器
web_extract(url="https://example.com", query="文章列表", selector="div.post-item")

# 增量更新（监控变化）
web_extract(url="https://example.com/news", query="最新消息", diff=true)
```

**注意事项**：
- 动态渲染页面（JavaScript 加载内容）可能需要配合 browser 工具
- 部分网站有反爬机制，提取可能失败
- 选择器语法支持 XPath 和 CSS，选择器为空时自动进行正文提取

## 2. browser（浏览器自动化）

**工具**：`browser_navigate`、`browser_snapshot`、`browser_click`、`browser_type`、`browser_scroll`、`browser_vision`、`browser_console` 等

### browser_navigate

**简介**：打开指定 URL，在浏览器中加载页面。是所有浏览器操作的第一步。

**常见用法**：
```python
browser_navigate(url="https://www.google.com")
```

**返回值**：页面快照，包含可交互元素及其引用 ID（格式为 `@e1`、`@e2` 等）。

---

### browser_snapshot

**简介**：获取当前页面的文本快照，展示可交互元素。配合 browser_click 和 browser_type 使用。

**参数**：
- `full=false`（默认）：紧凑视图，仅显示可交互元素
- `full=true`：完整页面内容

**常见用法**：
```python
browser_snapshot()           # 紧凑视图
browser_snapshot(full=true) # 完整内容
```

---

### browser_click

**简介**：点击页面上的元素。通过引用 ID（如 `@e5`）定位目标。

**常见用法**：
```python
browser_click(ref="@e5")  # 点击搜索按钮
```

---

### browser_type

**简介**：在输入框中输入文本。先清空输入框，再填入新文本。

**常见用法**：
```python
browser_type(ref="@e3", text="Hello World")
```

---

### browser_scroll

**简介**：滚动页面，揭示当前视口之外的内容。

**参数**：
- `direction="down"` 或 `"up"`

**常见用法**：
```python
browser_scroll(direction="down")
browser_scroll(direction="up")
```

---

### browser_vision

**简介**：对当前页面截图并用视觉 AI 分析。用于理解复杂布局、CAPTCHA、验证码等。

**常见用法**：
```python
browser_vision(question="页面上显示了什么错误信息？")
browser_vision(question="验证码是什么数字？", annotate=true)  # 标注交互元素
```

---

### browser_console

**简介**：获取浏览器控制台输出和 JavaScript 错误，用于调试动态网页问题。

**常见用法**：
```python
browser_console()                        # 获取所有日志
browser_console(clear=true)             # 获取并清除日志
browser_console(expression="document.title")  # 执行 JS 并获取返回值
```

---

### browser_back / browser_press

**简介**：
- `browser_back`：返回上一页
- `browser_press`：模拟键盘按键（Enter、Tab、Escape、ArrowDown 等）

**常见用法**：
```python
browser_back()
browser_press(key="Enter")
browser_press(key="Tab")
```

---

### browser_get_images

**简介**：获取当前页面所有图片的 URL 和 alt 文本，用于查找要分析的图片。

**常见用法**：
```python
browser_get_images()
```

## 3. terminal（终端与进程管理）

**工具**：`terminal`（执行命令）、`process`（管理后台进程）

### terminal

**简介**：在 Linux/macOS 终端执行 Shell 命令。文件操作请用 file 工具，不要用 `cat`/`head`/`tail`/`grep`/`find`/`ls`。

**核心概念**：
- **Shell**：命令解释器，接收用户输入并调用操作系统功能。常见 Shell：Bash（默认）、Zsh、Fish。
- **工作目录（cwd）**：命令执行的目录路径。相对路径以此为基准。
- **PTY（Pseudo-Terminal）**：伪终端，让交互式程序（如 vim、python REPL）认为自己在真实终端中运行。
- **Timeout**：命令最大执行时间（秒），超时后强制终止。

**参数说明**：
| 参数 | 类型 | 说明 |
|------|------|------|
| `command` | string | 要执行的命令 |
| `workdir` | string | 工作目录路径 |
| `timeout` | int | 超时时间（默认 180 秒） |
| `background` | bool | 后台模式，启动后立即返回，不阻塞 |
| `pty` | bool | PTY 模式，用于交互式程序 |
| `notify_on_complete` | bool | 任务完成时通知 |
| `watch_patterns` | list | 监控输出中的特定字符串并通知 |

**常见用法**：
```python
# 普通命令
terminal(command="ls -la", workdir="/home/ubuntu")

# 后台运行服务器
terminal(command="python server.py", background=true, notify_on_complete=true)

# 交互式程序（需要 PTY）
terminal(command="python", pty=true)

# 超时设置
terminal(command="sleep 30 && echo done", timeout=10)
```

**返回值**：`{"output": "...", "exit_code": N}`

**注意事项**：
- 禁止用 `cat` 读文件 → 用 `read_file` 替代
- 禁止用 `grep/rg/find` 搜索 → 用 `search_files` 替代
- 禁止用 `ls` 列目录 → 用 `search_files(target='files')` 替代
- 禁止用 `sed/awk` 编辑文件 → 用 `patch` 替代
- 交互式程序必须用 `pty=true`，否则会挂起

---

### process

**简介**：管理 terminal 启动的后台进程。需先用 `terminal(background=true)` 启动进程获得 session_id。

**参数说明**：
| 参数 | 类型 | 说明 |
|------|------|------|
| `session_id` | string | 进程 session ID（必填） |
| `action` | string | 操作类型 |

**action 类型**：
| action | 说明 |
|--------|------|
| `list` | 列出所有后台进程 |
| `poll` | 检查状态 + 获取新输出 |
| `log` | 获取完整输出（可分页） |
| `wait` | 阻塞直到进程结束 |
| `kill` | 终止进程 |
| `write` | 向 stdin 发送原始数据 |
| `submit` | 向 stdin 发送数据 + 回车 |
| `close` | 关闭 stdin（发送 EOF） |

**常见用法**：
```python
# 列出所有后台进程
process(action="list")

# 检查状态
process(action="poll", session_id="abc123")

# 获取输出
process(action="log", session_id="abc123", offset=-200, limit=100)

# 发送命令（回答交互式提示）
process(action="submit", session_id="abc123", data="y")

# 终止进程
process(action="kill", session_id="abc123")
```

## 4. file（文件操作）

**工具**：`read_file`、`write_file`、`patch`、`search_files`

### read_file

**简介**：读取文本文件内容，支持分页。替代 `cat`/`head`/`tail`。

**参数**：
| 参数 | 类型 | 说明 |
|------|------|------|
| `path` | string | 文件路径 |
| `offset` | int | 起始行号（1-indexed，默认 1） |
| `limit` | int | 最大行数（默认 500，最大 2000） |

**返回值**：包含 `content`、`total_lines` 的字典。

**常见用法**：
```python
read_file(path="/home/ubuntu/config.yaml")
read_file(path="/home/ubuntu/config.yaml", offset=100, limit=50)
```

**注意**：单次读取超过约 100KB 会被拒绝，用 offset/limit 分段读取。

---

### write_file

**简介**：写入内容到文件，完整覆盖原内容。替代 `echo`/`cat heredoc`。

**常见用法**：
```python
write_file(path="/home/ubuntu/test.txt", content="Hello World\n")
```

**注意**：会完整覆盖原文件，用 `patch` 进行局部修改。

---

### patch

**简介**：在文件中查找并替换指定内容。替代 `sed`/`awk`。

**参数**：
| 参数 | 类型 | 说明 |
|------|------|------|
| `path` | string | 文件路径 |
| `old_string` | string | 要查找的文本（必须唯一或配合 replace_all） |
| `new_string` | string | 替换后的文本 |
| `replace_all` | bool | 替换所有匹配（默认 false） |

**常见用法**：
```python
# 单次替换
patch(path="/home/ubuntu/config.yaml", old_string="port: 8080", new_string="port: 9090")

# 全部替换
patch(path="/home/ubuntu/config.yaml", old_string="debug: true", new_string="debug: false", replace_all=true)

# 删除内容（new_string 为空）
patch(path="/home/ubuntu/config.yaml", old_string="old_setting", new_string="")
```

**注意**：使用前需读取文件确认内容，确保 old_string 在文件中唯一。

---

### search_files

**简介**：搜索文件内容或按名称查找文件。替代 `grep`/`rg`/`find`。

**参数**：
| 参数 | 类型 | 说明 |
|------|------|------|
| `pattern` | string | 正则表达式（内容搜索）或 glob 模式（文件搜索） |
| `target` | string | `content`（搜索内容）或 `files`（按名称查找） |
| `path` | string | 搜索目录 |
| `file_glob` | string | 过滤文件类型，如 `*.py` |
| `output_mode` | string | `content`（显示匹配行）、`files_only`、`count` |
| `context` | int | 上下文的行数 |
| `limit` | int | 最大结果数 |

**常见用法**：
```python
# 搜索文件内容
search_files(pattern="def main", path="/home/ubuntu/project")
search_files(pattern="TODO", path="/home/ubuntu", output_mode="content", context=2)

# 查找文件
search_files(pattern="*.md", target="files", path="/home/ubuntu")
search_files(pattern="*config*", target="files", path="/home/ubuntu")
```

## 5. code_execution（代码执行）

**工具**：`execute_code`

### execute_code

**简介**：执行 Python 代码，可调用 Hermes 工具。适合需要多次工具调用和条件判断的工作流。

**核心概念**：
- **有状态执行**：代码在同一次执行中运行，多个工具调用之间变量持久化。
- **内置工具**：`from hermes_tools import terminal, read_file, patch, write_file, search_files`
- **内置函数**：`json_parse(text)`、`shell_quote(s)`、`retry(fn, max_attempts, delay)`

**限制**：
- 5 分钟超时
- stdout 最大 50KB
- 每次最多 50 个工具调用

**常见用法**：
```python
execute_code(code='''
from hermes_tools import terminal, read_file

# 读取文件
result = read_file(path="/home/ubuntu/config.yaml")
print(result["content"][:100])

# 执行命令
output = terminal(command="ls -la /home/ubuntu")
print(output["output"])
''')
```

**注意**：
- 适合 3+ 工具调用的复杂逻辑
- 简单单次工具调用直接用对应工具即可，无需 execute_code

## 7. image_gen（图像生成）

**工具**：`image_gen`（或调用外部工具）

### 常见图像生成工具

通过调用外部 API（如 DALL-E、Midjourney、Stable Diffusion）生成图片。

**核心概念**：
- **Prompt（图生图提示词）**：描述想要生成的图像的文本提示词，越详细越具体效果越好。
- **Negative prompt（反向提示词）**：描述不想出现在图像中的元素。
- **分辨率（Resolution）**：图像尺寸，如 512x512、1024x1024。
- **Seed（种子）**：随机种子，同一 seed + 相同参数会产生相同图像，用于复现。

**常见用法**：
```python
image_gen(prompt="A cute cat sitting on a windowsill, sunset lighting, photorealistic", size="1024x1024")
```

**返回**：图片路径或 URL，可直接发送给用户或保存到本地。

## 8. tts（语音合成）

**工具**：`text_to_speech`

### text_to_speech

**简介**：将文本转换为语音音频，返回本地文件路径。在支持语音的平台上自动作为语音消息发送。

**核心概念**：
- **TTS（Text-to-Speech）**：文字转语音技术。
- **Provider（提供商）**：TTS 服务商，不同提供商的音色、音质、费用不同。
- **字符限制**：不同 provider 有不同字符上限（Edge TTS 约 10000，ElevenLabs 5000-40000）。
- **语速/音调**：可调节语音的速度和音调。

**常见用法**：
```python
text_to_speech(text="你好，欢迎使用 Hermes Agent！")
text_to_speech(text="这是一个长文本..."，output_path="/home/ubuntu/voice.mp3")
```

**返回**：`MEDIA:/path/to/audio.mp3` 格式的本地文件路径。

**注意事项**：
- 超长文本会自动截断
- 需要配置 TTS provider（Edge TTS 免费开箱即用，ElevenLabs 需要 API Key）

## 10. todo（任务规划）

**工具**：`todo`

### todo

**简介**：管理当前会话中的任务列表。用于复杂多步骤任务的规划和进度跟踪。

**核心概念**：
- **有且只有一个进行中任务**：同一时间只能有一个 `in_progress` 状态的任务。
- **任务状态**：`pending`（待处理）、`in_progress`（进行中）、`completed`（已完成）、`cancelled`（已取消）。
- **merge 模式**：`merge=false`（默认）替换整个列表；`merge=true` 按 id 更新现有项并添加新项。

**常见用法**：
```python
# 读取当前任务列表
todo()

# 替换/创建任务列表
todo(todos=[
    {"id": "1", "content": "安装依赖", "status": "completed"},
    {"id": "2", "content": "配置数据库", "status": "in_progress"},
    {"id": "3", "content": "启动服务", "status": "pending"},
])

# 按 id 更新（不覆盖其他任务）
todo(todos=[
    {"id": "2", "content": "配置数据库（已完成）", "status": "completed"},
    {"id": "3", "content": "启动服务", "status": "in_progress"},
], merge=true)
```

## 11. memory（持久化记忆）

**工具**：`memory`

### memory

**简介**：保存持久化信息到跨会话记忆。保存后即使重启也依然存在。

**核心概念**：
- **目标**：`user`（用户信息，如偏好、习惯）和 `memory`（个人笔记，如环境事实、项目约定）
- **操作**：`add`（新增）、`replace`（替换）、`remove`（删除）
- **内容**：保存声明性事实，不保存任务进度或临时 TODO

**常见用法**：
```python
# 保存用户偏好
memory(action="add", target="user", content="用户使用简体中文回复")

# 保存环境事实
memory(action="add", target="memory", content="服务器系统为 Ubuntu 22.04")

# 更新已有内容
memory(action="replace", target="memory", content="新内容", old_text="旧内容")

# 删除
memory(action="remove", target="memory", old_text="要删除的内容")
```

**注意事项**：
- 不要保存任务进度、会话结果、已完成工作的日志
- 优先保存用户纠正、偏好、环境事实、工具约定等
- 任务完成后删除对应的临时记忆

## 12. session_search（会话搜索）

**工具**：`session_search`

### session_search

**简介**：搜索跨会话的历史对话记录。根据关键词查找过去讨论过的内容。

**核心概念**：
- **FTS5（Full-Text Search 5）**：全文搜索引擎，默认按 AND 匹配关键词。
- **布尔搜索**：用 `OR` 连接关键词扩大搜索范围（如 `elevenlabs OR baseten`）。
- **短语搜索**：用引号包裹精确短语（如 `"docker networking"`）。
- **前缀搜索**：用 `*` 匹配前缀（如 `deploy*`）。

**常见用法**：
```python
# 查看最近会话
session_search()

# 关键词搜索
session_search(query="docker networking")
session_search(query="elevenlabs OR baseten")
session_search(query="\"docker networking\"")

# 限制结果数
session_search(query="python", limit=5)
```

**返回值**：匹配会话的摘要列表，包含时间、来源、摘要。

## 13. clarify（用户交互）

**工具**：`clarify`

### clarify

**简介**：向用户提问以获取澄清、反馈或决策。用于任务不明确需要用户选择的情况。

**核心概念**：
- **多选模式**：提供 2-4 个选项，用户选择一项或输入自定义答案
- **开放模式**：不提供选项，用户自由输入
- **自动追加**：多选模式下会自动添加"其他（输入你的答案）"选项

**常见用法**：
```python
# 多选提问
clarify(question="你想用哪个模型？", choices=["Claude 3.5", "GPT-4", "Gemini Pro"])

# 开放提问
clarify(question="你想要什么样的输出格式？")
```

**注意事项**：
- 不要用于简单的是/否确认，直接执行即可
- 不要用于已经明确的决策，直接执行即可
- 适用于：任务不明确、结果需要用户确认、有多种可行方案

## 14. delegation（任务委托）

**工具**：`delegate_task`

### delegate_task

**简介**：生成独立的子 Agent 并行处理任务。子 Agent 有独立的对话、终端和工具集。

**核心概念**：
- **角色**：`leaf`（默认，默认工作者，不能再委托）、`orchestrator`（协调者，可继续委托）
- **工具集限制**：可指定子 Agent 使用的工具集（如 `["terminal", "file"]`）
- **上下文传递**：子 Agent 没有父会话的记忆，必须通过 `context` 传递所有必要信息
- **并发限制**：默认最多 3 个并发子 Agent

**常见用法**：
```python
# 单任务委托
delegate_task(goal="分析 /home/ubuntu/logs 下的错误日志，输出摘要", context="日志位于 /home/ubuntu/logs/*.log", toolsets=["terminal", "file"])

# 批量并行委托
delegate_task(tasks=[
    {"goal": "任务 A", "context": "...", "toolsets": ["terminal"]},
    {"goal": "任务 B", "context": "...", "toolsets": ["terminal"]},
    {"goal": "任务 C", "context": "...", "toolsets": ["terminal"]},
])
```

**返回值**：各子任务的结果数组。

**注意事项**：
- 子 Agent 结果是自我报告，不一定是经验证的事实
- 外部副作用操作（HTTP POST、远程写入）需要自己验证
- 不适合需要用户交互的子任务
- 长时间运行任务用 `cronjob` 而非 `delegate_task`

## 15. cronjob（定时任务）

**工具**：`cronjob`

### cronjob

**简介**：创建、管理定时任务，让任务在指定时间自动运行。

**核心概念**：
- **Schedule（调度表达式）**：常用格式 `30m`（30分钟后）、`every 2h`（每2小时）、`0 9 * * *`（每天9点）、ISO 时间戳
- **Delivery（投递目标）**：`origin`（当前聊天）、`local`（仅存本地文件）、`平台:chat_id`（指定聊天）
- **Skills 附加**：定时任务可以预加载技能，确保任务执行时加载正确的上下文
- **脚本注入**：通过 `script` 参数在任务运行前执行脚本，其 stdout 注入到 prompt

**常见用法**：
```python
# 创建定时任务
cronjob(action="create", prompt="检查服务器状态，输出到 ~/status.txt", schedule="every 1h", name="健康检查")

# 列出所有任务
cronjob(action="list")

# 暂停任务
cronjob(action="pause", job_id="abc123")

# 恢复任务
cronjob(action="resume", job_id="abc123")

# 删除任务
cronjob(action="remove", job_id="abc123")

# 立即触发
cronjob(action="run", job_id="abc123")
```

**注意事项**：
- 定时任务在新会话中运行，没有当前聊天上下文
- prompt 必须完全自包含，所需信息全部在 prompt 中说明
- 不要在定时任务中递归创建更多定时任务

## 16. messaging（跨平台消息）

**工具**：`send_message`

### send_message

**简介**：向已连接的 messaging 平台发送消息，或列出所有可用目标。

**核心概念**：
- **Home Channel（主频道）**：默认消息发送目标，当前会话的来源平台
- **平台格式**：`telegram`、`telegram:chat_id`、`discord:#channel`、`discord:chat_id:thread_id`、`slack:#engineering`、`weixin`、`weixin:chat_id` 等
- **Media 发送**：在消息中包含 `MEDIA:/path/to/file` 可发送图片/文件

**常见用法**：
```python
# 列出所有可用目标
send_message(action="list")

# 发送到主频道
send_message(message="服务器已启动！")

# 发送到指定平台/频道
send_message(target="telegram:123456789", message="Hello from Hermes!")
send_message(target="discord:#engineering", message="Build completed!")

# 发送图片
send_message(message="截图如下：\nMEDIA:/home/ubuntu/screenshot.png")
```

**注意事项**：
- 用户明确指定频道时才需要 list 后再发送
- 直接说"发送到 telegram"会直接发给主频道，无需 list
